# Part3 AI总结以上各页面和说法的要素 

前端设计要素全景总结
一、基础可用性层（硬指标，底线）
要素	核心要点
布局完整性	无溢出、重叠、断裂；Flex/Grid 正常渲染
可读性	对比度 ≥ WCAG AA（4.5:1）；字号 ≥ 12px；行高 ≥ 1.4
响应式	375px / 1440px 宽度下均无水平滚动条
性能	无白屏闪烁；FCP < 1.5s（WPO 优化）
可访问性（A11y）	语义化标签、键盘导航、ARIA 属性、焦点管理、表单可访问性（WCAG 2.2）
语义化/SEO	合理的 Markup 结构和 Meta-data
二、视觉协调性层（规则可编码）
要素	核心要点
色彩和谐度	遵循色彩理论（互补/类比/三角色）；主色 ≤ 3 种；色板统一
间距一致性	8px 网格系统；同类元素间距方差小；padding/margin 有规律
字体层级	字体族 ≤ 2；h1-h6 字号层级清晰递减；杜绝"全是同一字号"
视觉重心	首屏有明确主导元素；避免"到处都在抢注意力"
设计令牌（Design Tokens）	统一颜色、字号、间距、圆角、阴影等规范
三、设计感层（需审美判断）
要素	核心要点
留白运用	恰当使用负空间；不拥挤不空洞；层次感
设计原创性	跳出"AI slop"模板感（蓝紫渐变 + 圆角卡片 + emoji icon 的千篇一律）
品牌感/调性	统一的视觉语言和氛围，而非组件堆砌
视觉动线	引导用户视线流动的路径设计
"做减法"	识别并移除冗余元素、合并相似功能、精简信息层级、突出核心路径
四、交互与动效层（锦上添花）
要素	核心要点
微交互自然度	hover/click 过渡流畅；动画时间曲线合理
有意义的动效	动效服务于交互（按钮反馈、模态框过渡、加载动画、状态切换），非无意义炫技
像素级细节	统一间距、修正对齐、优化阴影圆角、强化视觉层次——这"最后 10%"决定业余 vs 专业
五、流程与工程要素
要素	核心要点
评判方式分层	能写成确定性规则的 → 机判；规则模糊但异常可检出 → 机判筛+人确认；需文化/审美语境 → 纯人判
偏好对而非绝对打分	人对"A vs B 哪个更好"的判断远比对"这个打几分"可靠；多人独立判断 + 一致性检验（Fleiss' Kappa）
防 LLM 循环	Ground truth 不能来自 LLM 打分——L1/L2 用确定性规则，L3/L4 必须以人类标注为唯一真值
防多样性坍缩	Prompt 变异矩阵（风格×行业×色调×复杂度）；反模板约束；embedding 相似度监控；保留"奇怪但有趣"的样本
专业评测维度参考	Design、Usability、Creativity、Content 四大主维度，辅以 Animations、Accessibility、WPO、Responsive、SEO 等次要维度

- 色板/色相分析
- 间距一致性/8px网格

- 元素可访问性

- 溢出、重叠检测 https://docs.pingcode.com/baike/2518573
- FCP 性能指标
- 动效时长，特定要素真正可见所用的时间
- *新增* 浏览器编译报错情况

考虑专门构建评定模型的维度
即，专门收集相关数据、训练一个和这个域相关的评价模型
- 配色协调度        提取优质设计网站配色模式为正例，随机使用配色为负例，训练一个评定模型
- 视觉重心、层次感    训练“提取视觉核心要素”的模型，用来提取实际网站表现视觉上的重心，并与基于纯文本/意图的、仅以网页设计目标和网页代码（不含CSS）输入判断重心元素的模型，比对相合度
- 字体搭配          大小接近配色的判定法；层次接近后者的判定法。
- AI slop 模板感    用当前流行的设计AI（尤其是确实有slop黑名的）直出一些网站，和真实网站分别做AI判定能力的正负例。不过用的时候还是按置信度转化为评分的模式。

调一个尽量与人对齐的AI prompt后直接评分：
- 组件冗余/需减法
- 动效意义性
- 整体美学综合
- *新增* 设计风格简约程度


其余要素直接要求AI判定，看看它们对相关要素比较差的正负例的判断能力。如果还可以就加上，如果不太行就暂时不做。如果有迭代需求，会申请以人工标注为主去做数据。

特别的，还要迭代一个对定制性要求的遵循能力：在常规的、无限制的“生成XX主题的网站”类数据中混入一些“指定需求，例如要求一定得是特定配色，某某LOGO一定要大，某个元素必须要显眼。即使这种要求违背美学指标。主要关切指令遵循的能力。

# Part4 阈值判定
收集一些未经专门调整的，甚至就是客户说明存在问题的AI生成的结果，形成集A；收集实际的人工作的高质量网站（比如刚才的awwwards的高分网站）形成集B；收集一些拥有较多用户、大规模使用的网站，尤其是年限较长的网站（无论本身设计的好不好）形成集C；人工查看当前的规则在三个集上评分的表现、分布，确定判定能力/误判比例/合适的阈值。
