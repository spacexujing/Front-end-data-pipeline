# 关键决策：
1. 主要迭代方向、指标的选择。
来源于AI方案、网络相关信息教学、其它专业设计评分机构指标。
AI和网络教学能够给出一些非常客观和具体的设计思路说明，专业机构的评分方案则很好地给出了一个设计的美学涉及哪些主观方向。
当然，实际此处应该咨询专家，尤其是那些所谓的客观设计思路，需要更高可信度的意见来评价到底客不客观。

2. 评测方案的选择：
对迭代方向和指标的思考。所有可以规则化的，只要合适都可以放入，而偏主观的就只能交由AI打分或者人工。
对于一些还算有规律可循的主观项目，如组件冗余性，可以交给AI，但对于主观性特别强、特别笼统的方向，尤其是最纯正的“美学”“创新性”之类的，则交给人工做GRPO标注。

3. 数据设计的选择：
- 最常规的完整网页，使用合成的高分数据或直接使用清洗后的现成网页数据。直接SFT。最常规的方案，训练基础的“写出好网页”的能力。
- GRPO训练。因为很多美学要素的可描述性实在太差了，很难由评分机制去良好区分、训入模型。因此使用GRPO方案。
- 特定能力的训练。 很多规则化的内容（例如配色、字体等）是仅关联CSS文件的，可以以较小的成本去大量合成，而且预期会能带来比较明显的提升。尤其是，如果确实某方面存在比较明确的“设计准则”，那么在后训练中特化训练这一部分能很好地提升模型相关的能力。并可以用DPO进一步强化。因为本就是遵循明确的规则，所以不太需要担心刻板化问题。
- DPO数据，根据实际观察的模型情况，压制特定行为（尤其题设中存在已知的负面行为）。
- “指定模式更改”能力。SFT多轮数据。我认为这是实际使用场景中必然高频出现的情况，“甲方”对整体满意，但是对一些细节有意见/有新的想法。模型必须有相关能力。       因为强相关历史上文和本轮问询，一般而言SFT效果即足够。

# 什么地方没有用AI？为什么
1. 一些简单的过渡代码，尤其是pipeline.ipynb里的那些。纯粹是描述给AI并不比我自己写快。
2. pipeline.ipynb中能看到的基础的prompt。最初始的版本其实还是人给比较好，能根据经验引入很多先验知识。实际工作的话，此处应该引入AI迭代，但时间所限，确实也就没有余力像实际工作一样真正好好迭代prompt，所以这环节暂时没有AI参与。

# AI 在哪掉坑？如何解决
1. AI的输出稳定性和指令遵循能力是有缺陷的。尤其是在html输出这一侧，它喜欢在html代码前面加一些有的没的的文本，即使要求不要加也会有这个问题。（估计是训练数据有点刻板化行为）。解决方法是改用json输出。json输出的分离性一般是相当良好的，基本能保证该是code的地方确实有且只有code。
2. 其实AI给的评价阈值还是很AI。我发现这个问题了，但暂时没法解决，需要专家介入重构阈值和提示词。
3. token长度限制导致的网页输出不完整。调高了。如果是实际工作的话，其实一般在大规模生产前会先生成十来条人工确认一下效果，此类问题就能被解决。实际上这次的十来条样例数据本身也起到了同样的作用。
4. 对模拟后网页打开后的自动化评测。在方案中有相关内容，但没有相关代码流程，因为试验后完全不可用。这可能需要一个专门的agent能力或者skill，以及一定的多模能力。要实际使用得花时间研究一下。

# 最可能失败的地方、如何迭代
1. 实际上，指标的划分可以分为两部分，一部分是“不做错事”，另一部分是“做对的事”。规则的引入很可能可以让模型少做违反设计规范的“错事”，如果原本的模型有这方面的问题，那我比较有信心。但真正意义上的美学能力的突破——几乎仅靠依赖于获取到的“高分设计”的水平，以及GRPO标注的效果。
而，GRPO标注一般很难由真正意义上的专家进行，或者说成本很高。有概率出现多次反复，比如先找了相对水平比较低的人，但是标注效果一般，有数量众多的循环胜利GRPO(A>B>C>A)，只好提高成本找更专业的人，如此反复几次的可能。
2. 我不确定遵守特定的、严明的“设计规范”是否合适。真的需要咨询专家。而且我觉得这有概率引入新的“AI味”。
3. 当前这个方案至少在设计上是试图通用性地应对所有类型的网页设计、整体性提升效果的。但我觉得不同主题的网页，美学评价指标可能大相径庭，甚至偶尔是相反的——我很难想象一个如互联网公司网站一般酷炫的政府网站/传统实业网站，前者的酷炫在后者的场景下反而感觉很虚浮，哪怕实际上要展示的标签和分类都是差不多甚至可能是完全相同的。     可能最终还是会有回归特定域做精细后训练的需要。
4. 根据用户的要求去正确微调当前设计在To C上是非常关键的能力。甚至可能比模型本身的审美能力更重要。当前它只被放在次要地位。这很可能需要专门的迭代。
5. 什么是“AI味”？AI味的设计本身并不劣质，甚至可能很优秀。我仍记得当初最早一批AI网页出来的时候……那就是当时最潮最炫的设计风格，是互联网企业和奢侈品网页真的会用的设计。……无非大家看这个味看得太多了有点腻。这可能不是从一般美学指标层面上可以排除的。而去AI味之后，又必须比“AI味”真的更好看而不是更难看。这个要求相当高，可能需要一套专门的“去AI化”的流程，学习相当前卫的设计样例，而且可能只能是个持续迭代的过程。时尚永远是变化的。
