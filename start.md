课后题（从 0 探索：Web 开发美学数据管线）

背景：客户模型能根据 prompt 生成网页 UI 代码，但产出质量参差——有的精致，很多是「AI slop」（一眼假、模板化、没设计感）。客户想让我们生产数据，用来评测、并最终训练模型产出更有美学质量的前端。

「美学好不好」是个主观、模糊的东西。这道题不给你任何评分标准、失败模式清单或样本，怎么把它变成可评、可复现、可规模化的数据，是要你从 0 探索的。

你的任务：

1. 探索 + 出方案

- 你怎么定义「web 前端的美学质量」？拆成哪些可操作的维度？
- 这些维度里，哪些能机器自动判定、哪些必须靠人/专家判？边界怎么划、为什么？
- 数据长什么样：打分？偏好对（A vs B）？带 rubric 的评判？你选哪种、为什么？
- ground truth 从哪来——你怎么避免"让一个 LLM 生成、再让一个 LLM 打分、就当成真值"这种循环？
- 怎么防生成样本的多样性坍缩？
- 最终交付给客户的东西长什么样（train 数据 + 评测集）？


2. 带 AI 落地，把方案的核心跑通
- 用 Claude Code / Codex，把你上面设计的管线做出一个能跑的最小版本：从 UI 生成 prompt 进去，到候选网页产出、按你的判据评判、最终落成结构化数据出来，整条链路能真的跑起来。最终交付一个小而完整的数据成品：
- 一套你自己定义的 rubric / 判据（评分维度 + 标准）；
- ≥3 条按这套判据、由管线可复现地产出的 data sample（不是手工拍的分）；
- 跑通整条链路的代码


3. 一页说明

- 你的关键决策（判据怎么来的、为什么这么拆）；
- 哪些交给了 AI、怎么交的、AI 在哪掉坑、你怎么发现并纠正；
- 你这套方案最可能在哪里失败、你会怎么迭代。
